# Review: `working-with-databases.md`

**Tier:** Concept page (`docs/getting-started/`, sibling to `working-with-documents.md` / `workflows.md`). Audience: a competent developer new to Primitive.

**Verdict:** The page opens correctly and the first half is a solid, well-ramped concept page — but the second half drifts to agent-guide altitude (error codes, per-phase timing internals, boolean-gate truth tables, truncation loop semantics) that already lives in `AGENT_GUIDE_TO_PRIMITIVE_DATABASES`, and it carries three concrete never-say violations. The fix is to trim depth and relocate reference material, not to restructure the ramp wholesale.

## Lenses that came back clean

- **Example parity:** Two `lone-js` blocks. Line 426 (`client.databases.subscribe` block) is a core-usage example with no Swift counterpart — but per finding #1 the Swift API genuinely doesn't exist, so the JS-only block is the correct treatment *once the narration is removed*; no corpus pair is required (file the bug instead). Line 380 (the `status: "open" | "in-progress" | "closed"` TS-union snippet) is a TypeScript-codegen-specific illustration where Swift adds nothing — acceptable lone block per STYLE.md. No parity action needed.
- **Facts (spot-verified):** `executeOperation`, `executeBatch`, `importCsv`, `timing`/`_timing`, the `databaseType` filter on `list()`, and the CEL functions (`isMemberOf`, `memberGroups`, `hasRole`, `fromWorkflow`) all verify against `library_repos/js-bao-wss/src/client/api/databasesApi.ts` and the server controllers. `databases.list()` owner/manager-only behavior and `databases.get()` any-authenticated behavior match the source. No fictional APIs found. (Finding #3 is a safety-qualification issue, not a wrong fact.)
- **Ramp (first half):** Concept → Key Properties → Quick Start → Operation Types is a clean monotonic ramp at the right entry altitude. The altitude problems (#4–#7) are depth-ceiling violations, not ordering violations — nothing advanced sits *above* something basic.
