# Review: `docs/index.md` (landing page snapshot)

**Tier: Landing / overview.** Per STYLE.md the entry altitude is "capabilities and value — what you get," and the hard ceiling is "never name mechanisms (file formats, codegen commands, schema syntax, specific API shapes)."

**Verdict:** This page is over-scoped for its tier. It breaches the landing ceiling repeatedly — schema-definition and client-construction code, an infrastructure architecture diagram, and a "what runs where" mechanism table all sit on the overview page — and its framing is browser/web-only despite the platform being multi-platform (web + iOS). The prose is otherwise clean and concrete in places, but the structural violations require a real restructure, not in-place edits.

## Notes for the author

The current published `docs/index.md` already demonstrates the target shape (a "From Idea to Production" lifecycle list, differentiator-led "Why Primitive?", capability sections, no diagram, no code, no client/server table) and is the reference to converge on.
